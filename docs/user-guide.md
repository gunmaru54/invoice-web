# 견적서 관리 서비스 사용 가이드

이 가이드는 Notion을 사용하여 견적서를 작성하고 고객과 공유하는 방법을 안내합니다.

---

## 1. 서비스 흐름

Notion에서 견적서 작성 → 어드민에서 URL 복사 → 고객에게 URL 전달 → 고객이 웹에서 열람 및 PDF 저장

---

## 2. Notion 견적서 작성 방법

### 필수 속성 목록

> ⚠️ 아래 속성명을 임의로 변경하면 서비스가 정상 동작하지 않습니다.

| 속성명 | 타입 | 예시 | 설명 |
|--------|------|------|------|
| `Name` | Title | `홍길동 웹사이트 제작 견적` | 견적서 제목 |
| `client_name` | Text | `홍길동` | 고객 이름 |
| `client_email` | Email | `hong@example.com` | 고객 이메일 (선택) |
| `issue_date` | Date | `2024-01-15` | 견적서 발행일 |
| `expiry_date` | Date | `2024-02-15` | 견적서 유효기간 (선택) |
| `status` | Select | `draft` | 견적서 상태 (아래 참고) |
| `total_amount` | Number | `1500000` | 총 견적 금액 (원 단위, 숫자만) |
| `note` | Text | `부가세 포함` | 비고 사항 (선택) |

> 공개 URL 식별자(slug)는 별도 속성 없이 Notion 페이지 ID를 그대로 사용합니다. 어드민의 **URL 복사** 버튼을 통해 자동으로 생성됩니다.

### 견적서 상태 5종

| 상태 값 | 화면 표시 | 사용 시점 |
|---------|-----------|-----------|
| `draft` | 초안 | 작성 중인 견적서 |
| `sent` | 검토 중 | 고객에게 URL을 전달한 후 |
| `viewed` | 열람됨 | 고객이 URL을 열람한 것을 확인한 후 |
| `accepted` | 수락됨 | 고객이 견적을 수락한 후 |
| `rejected` | 거절됨 | 고객이 견적을 거절한 경우 |

---

## 3. 견적 항목(items) 추가 방법

견적 항목은 Notion의 별도 items 데이터베이스에 추가하고, 견적서 페이지와 Relation으로 연결합니다.

### items DB 속성

| 속성명 | 타입 | 예시 | 설명 |
|--------|------|------|------|
| `item_name` | Title | `웹 퍼블리싱` | 견적 항목 이름 |
| `qty` | Number | `1` | 수량 |
| `unit_price` | Number | `500000` | 항목 단가 (원 단위) |
| `amount` | Number | `500000` | 수량 × 단가 (수동 입력) |
| `note` | Text | `디자인 시안 3회 수정 포함` | 항목 설명 (선택) |

> ⚠️ **금액 필드는 자동 계산되지 않습니다.** 수량 × 단가를 직접 입력하세요.

---

## 4. 고객에게 견적서 공유하기

1. 어드민 대시보드(`{APP_ORIGIN}/invoices`)에서 해당 견적서를 찾습니다.
2. **URL 복사** 버튼을 클릭합니다.
3. 복사된 URL(`{APP_ORIGIN}/quote/{pageId}`)을 이메일이나 메신저로 전달합니다.
4. Notion에서 해당 견적서의 `status`를 `sent`로 변경합니다.

> ⚠️ **URL을 직접 타이핑하지 마세요.** 반드시 어드민의 URL 복사 버튼을 사용하세요.

## 5. 자주 묻는 질문 (FAQ)

**Q. Notion에서 수정했는데 웹에 반영이 안 됩니다.**
A. 캐시 적용으로 최대 60초 지연될 수 있습니다. 1분 후 새로고침하세요.

**Q. 고객이 "잘못된 주소입니다" 페이지가 나온다고 합니다.**
A. 어드민 URL 복사 버튼으로 복사한 URL인지 확인하세요. Notion 페이지가 삭제되거나 이동되면 기존 URL은 더 이상 유효하지 않습니다.

**Q. PDF에서 한글이 깨져 보입니다.**
A. 브라우저를 최신 버전으로 업데이트 후 다시 시도하세요. 문제가 지속되면 개발자에게 문의하세요.

**Q. 속성 이름을 변경해도 되나요?**
A. **안 됩니다.** 속성명이 코드에 하드코딩되어 있어 변경 시 데이터가 표시되지 않습니다.

**Q. 견적서 항목이 화면에 표시되지 않습니다.**
A. Notion items DB에서 해당 견적서 페이지와 Relation이 올바르게 연결되어 있는지 확인하세요. `item_name`이 비어 있는 항목은 자동으로 무시됩니다.

**Q. 여러 고객에게 같은 견적서를 보내도 되나요?**
A. 동일한 URL을 공유하면 됩니다. 내용을 다르게 하려면 Notion에서 페이지를 복제하여 새 견적서를 만드세요. (복제 시 새 페이지 ID가 발급되어 새 URL이 생성됩니다)

## 6. 장애 대응 체크리스트

증상이 발생하면 아래 순서대로 확인하세요.

| 증상 | 확인 사항 |
|------|-----------|
| 대시보드에 데이터가 없음 | 1) Notion DB에 데이터 있는지 확인 2) NOTION_API_KEY 유효한지 Vercel 환경 변수 확인 |
| 공개 URL에서 404 | 1) 어드민 URL 복사 버튼으로 생성한 URL인지 확인 2) Notion 페이지가 삭제되지 않았는지 확인 |
| PDF 한글 깨짐 | 브라우저 캐시 삭제 후 재시도, 최신 Chrome 사용 권장 |
| 사이트 전체 오류 | Vercel 대시보드 → Deployments에서 최신 배포 로그 확인 |

### 개발자 문의 시 포함 사항

- 오류 발생 일시
- 오류 화면 스크린샷
- 접속 URL
- Notion 페이지 링크 (있는 경우)
