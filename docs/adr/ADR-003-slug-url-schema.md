# ADR-003: slug URL 스키마 및 공개 URL 구조

| 항목 | 내용 |
|------|------|
| 상태 | 채택(Accepted) |
| 작성일 | 2026-04-23 |
| 결정자 | 개발팀 |

---

## 배경

공개 견적서 뷰어 URL(`/quote/{slug}`)의 `slug` 값을 어떻게 결정할지 선택이 필요했다.

후보:
1. Notion 페이지 ID (UUID) 직접 사용
2. 사람이 읽기 쉬운 커스텀 slug (예: `hong-website-2024-001`)
3. 별도 순번 생성 (예: `INV-001`)

---

## 결정

**Notion 페이지 ID(UUID)를 slug로 직접 사용한다.**

공개 URL 형식: `{APP_ORIGIN}/quote/{notion-page-id}`

예시: `https://invoice.example.com/quote/a1b2c3d4-e5f6-4a7b-8c9d-0e1f2a3b4c5d`

---

## 근거

- **전역 유일성 보장**: Notion이 페이지 생성 시 UUID를 자동 발급하므로 중복 없음
- **운영 복잡도 최소화**: 별도 slug 관리 테이블이나 중복 검사 로직 불필요
- **즉시 공유 가능**: Notion 페이지 생성 즉시 URL 활성화, 추가 설정 없음
- **1인 사업자 MVP 적합**: 가독성보다 안정성이 우선인 MVP 단계

---

## 제약 및 트레이드오프

- **가독성 낮음**: UUID는 사람이 외우거나 타이핑하기 어렵다. 단, 어드민에서 URL 복사 버튼을 제공하므로 직접 입력 시나리오는 없음
- **Notion 페이지 삭제 시 URL 무효화**: 페이지 삭제 또는 이동 시 기존 URL이 404가 됨
- **URL이 예측 가능하지 않음**: 보안 관점에서 UUID는 추측이 어려워 오히려 유리

---

## 운영 규칙

1. URL 공유는 **반드시 어드민의 URL 복사 버튼** 사용
2. Notion 페이지 ID는 URL에 그대로 노출되므로 **페이지 삭제 금지** (삭제 시 고객 URL 무효화)
3. 견적서 복제 시 **새 UUID가 발급되어 새 URL이 생성됨** — 기존 URL은 유지됨

---

## 향후 계획 (v1.1)

- 사람이 읽기 쉬운 커스텀 slug 지원 검토 (예: `client-project-YYYYMM-001`)
- slug 변경 이력 관리 및 구 URL 리다이렉트 지원 검토
