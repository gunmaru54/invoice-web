# 노션 기반 견적서 웹 뷰어 MVP PRD

## 🎯 핵심 정보

**목적**: 사업자가 Notion에 입력한 견적서를 고객이 고유 URL로 웹에서 확인하고 PDF로 다운로드할 수 있도록 한다.
**사용자**: 견적서를 작성하는 1인 사업자(어드민)와 견적서를 확인하는 고객(클라이언트).

---

## 🚶 사용자 여정

### 어드민(사업자) 플로우

```
[Notion 데이터베이스에 견적서 입력]
        │
        ▼
[견적서 레코드 저장 → slug/ID 자동 생성]
        │
        ▼
[어드민 대시보드 접속] ──→ [견적서 목록 페이지]
        │                          │
        ▼                          ▼
[공유 URL 복사]           [견적서 상세 미리보기]
        │
        ▼
[클라이언트에게 URL 전달 (이메일/메신저)]
```

### 클라이언트(고객) 플로우

```
[공유 URL 수신]
        │
        ▼
[견적서 뷰어 페이지 진입]
        │
   ┌────┴────┐
   ▼         ▼
[웹에서    [PDF 다운로드
 견적서     버튼 클릭]
 확인]          │
               ▼
         [PDF 파일 저장]
```

### 페이지 전환 조건

| 출발 상태 | 조건 | 도착 페이지 |
|---|---|---|
| 어드민 대시보드 | 견적서 목록 클릭 | 견적서 목록 페이지 |
| 견적서 목록 | 특정 견적서 클릭 | 견적서 상세 미리보기 페이지 |
| 견적서 목록 | 공유 URL 복사 버튼 클릭 | (클립보드 복사 후 동일 페이지 유지) |
| 공유 URL 접속 | 유효한 slug | 견적서 뷰어 페이지 |
| 공유 URL 접속 | 존재하지 않는 slug | 404 에러 페이지 |
| 견적서 뷰어 | PDF 다운로드 버튼 클릭 | (PDF 파일 다운로드 트리거) |

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|---|---|---|---|---|
| F001 | Notion 견적서 목록 조회 | Notion 데이터베이스에서 전체 견적서 목록을 불러와 표시 | 어드민이 관리할 견적서 목록의 기반 | 견적서 목록 페이지 |
| F002 | Notion 견적서 상세 파싱 | Notion 페이지의 속성(제목, 금액, 항목, 고객명 등)을 파싱하여 구조화된 데이터로 변환 | 웹 뷰어 및 PDF 생성의 핵심 데이터 소스 | 견적서 상세 미리보기 페이지, 견적서 뷰어 페이지 |
| F003 | 견적서 웹 뷰어 렌더링 | 파싱된 견적서 데이터를 웹 페이지에 정형화된 UI로 렌더링 | 클라이언트가 웹에서 견적서를 확인하는 핵심 기능 | 견적서 뷰어 페이지 |
| F004 | 공유 URL 생성 및 복사 | 각 견적서에 대한 고유 공유 URL을 생성하고 클립보드에 복사 | 클라이언트에게 견적서를 전달하는 유일한 수단 | 견적서 목록 페이지 |
| F005 | PDF 다운로드 | 견적서 뷰어 페이지에서 PDF 파일 생성 후 다운로드 | 클라이언트가 견적서를 보관·출력하기 위한 필수 기능 | 견적서 뷰어 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|---|---|---|---|---|
| F006 | 어드민 대시보드 | 어드민 진입점, 견적서 목록 페이지로 이동하는 네비게이션 제공 | 어드민 작업 흐름의 시작점 | 어드민 대시보드 페이지 |
| F007 | 견적서 상세 미리보기 | 어드민이 공유 전에 견적서 렌더링 결과를 확인 | 공유 전 오류 확인을 위한 QA 기능 | 견적서 상세 미리보기 페이지 |
| F008 | 404 에러 처리 | 존재하지 않는 slug 접근 시 에러 페이지 표시 | 잘못된 URL 접근 시 사용자 경험 보호 | 404 에러 페이지 |

### 3. MVP 이후 기능 (제외)

- 견적서 승인/거절 기능 (클라이언트 서명)
- 이메일 자동 발송
- 견적서 만료일 설정 및 접근 제한
- 어드민 로그인/인증
- 견적서 열람 횟수 추적
- 다국어 지원

## 📄 페이지별 상세 기능

### 1. 어드민 대시보드 페이지

| 항목 | 내용 |
|---|---|
| **역할** | 어드민이 시스템에 진입하는 메인 허브 |
| **사용자 행동** | 견적서 목록 페이지로 이동 |
| **진입 조건** | 어드민 전용 URL 직접 접속 |
| **기능 목록** | 견적서 목록 페이지 이동 버튼, 총 견적서 수 요약 표시 |
| **구현 기능 ID** | F006 |

### 2. 견적서 목록 페이지

| 항목 | 내용 |
|---|---|
| **역할** | 어드민이 Notion에 등록된 모든 견적서를 조회하고 공유 URL을 관리 |
| **사용자 행동** | 견적서 목록 확인, 특정 견적서의 공유 URL 복사, 견적서 상세 미리보기로 이동 |
| **진입 조건** | 어드민 대시보드에서 견적서 목록 버튼 클릭 |
| **기능 목록** | Notion 연동 견적서 목록 표시(고객명·금액·날짜), 공유 URL 복사 버튼, 견적서 상세 미리보기 이동 링크 |
| **구현 기능 ID** | F001, F004 |

### 3. 견적서 상세 미리보기 페이지

| 항목 | 내용 |
|---|---|
| **역할** | 어드민이 클라이언트에게 공유하기 전 견적서 렌더링 결과를 확인 |
| **사용자 행동** | 견적서 내용 확인, 목록으로 돌아가기 |
| **진입 조건** | 견적서 목록 페이지에서 특정 견적서 클릭 |
| **기능 목록** | Notion 견적서 데이터 파싱 후 렌더링, 견적서 항목·금액·고객 정보 표시, 목록으로 돌아가기 버튼 |
| **구현 기능 ID** | F002, F007 |

### 4. 견적서 뷰어 페이지

| 항목 | 내용 |
|---|---|
| **역할** | 클라이언트가 공유 URL로 견적서를 확인하고 PDF로 다운로드 |
| **사용자 행동** | 견적서 내용 열람, PDF 다운로드 버튼 클릭 |
| **진입 조건** | 유효한 slug가 포함된 공유 URL 직접 접속 |
| **기능 목록** | 견적서 데이터 파싱 및 정형화된 UI 렌더링, 견적서 항목 테이블(품목·수량·단가·금액), 합계·세금·총액 표시, PDF 다운로드 버튼 |
| **구현 기능 ID** | F002, F003, F005 |

### 5. 404 에러 페이지

| 항목 | 내용 |
|---|---|
| **역할** | 존재하지 않는 URL 접근 시 사용자에게 안내 메시지 제공 |
| **사용자 행동** | 에러 메시지 확인 후 홈으로 이동 |
| **진입 조건** | 존재하지 않는 slug로 견적서 뷰어 URL 접속 |
| **기능 목록** | "견적서를 찾을 수 없습니다" 안내 메시지, 홈(어드민 대시보드) 이동 링크 |
| **구현 기능 ID** | F008 |

## 🗄️ 데이터 모델

> Notion 데이터베이스를 주 저장소로 사용합니다. 별도 DB는 MVP에서 사용하지 않습니다.

### Notion 견적서 데이터베이스 (Invoice)

| 필드명 | 설명 |
|---|---|
| title | 견적서 제목 |
| slug | 공유 URL용 고유 식별자 |
| client_name | 고객(클라이언트) 이름 또는 회사명 |
| issue_date | 견적서 발행일 |
| total_amount | 견적서 총 금액 |

### Notion 견적서 항목 (InvoiceItem)

> 견적서 페이지 내 하위 블록(테이블 또는 리스트)으로 표현

| 필드명 | 설명 |
|---|---|
| item_name | 품목명 |
| quantity | 수량 |
| unit_price | 단가 |
| amount | 금액 (수량 × 단가) |
| note | 비고 |
